Introduction

CodeIgniter Wizard is a PHP developer utility that helps quickly generate an advanced starting point for MySQL database-driven web applications with administrative interfaces by developing the most commonly used boilerplate code for CRUD operations and HTML files.

Using its database-first approach, you can define a database connection, select the tables and fields you want to include in your application, and then the wizard automatically generates the necessary models, views, and controllers - even the new CodeIgniter 4 entities, supporting standard database CRUD operations, that is, create, read, update and delete. It even automatically populates the sidebar and (dashboard) home pages with some basic data from your modules.

CodeIgniter Wizard generates most of the time-consuming boilerplate code needed for the basis of the backend of most web applications which will save you weeks if not months! You can then take the generated project and keep developing using the base code.

Most important of all, you are not bound to any CodeIgniter Wizard-specific libraries or base classes to further develop your application. Every code file CodeIgniter Wizard generates is standards-based, generic, secure, re-useable, yet simply replaceable.

What's New in CodeIgniter Wizard?

Below is a list highlighting the important changes in CodeIgniter Wizard across new versions.

  • Version 3 brings Codeigniter Shield as the official 'auth' library for authentication and authorization, providing an alternative to the older Myth-Auth project.
  • With version 3, Bootstrap 5-based templates have been revamped to support dark mode.
  • With version 2.5, the intrinsic database driver has been replaced with a more versatile one, adding support for MySQL versions higher than 8.0 as well as MariaDB, alleviating incompatibilities with newer authentication mechanisms such as sha256_password and cached_sha256_password.
  • As of version 2.1, the ability to add automatic UUID generation traits to your projects has been made possible.
  • Starting with version 2.0 version of CodeIgniter Wizard, automatically add user authentication and authorization for your CodeIgniter projects. 
  • CI Wiz is in English, but you can create localized web applications with an unlimited number of languages and translations.
  • Ability to export CSV and Excel files from list views
  • Many more... (See What's New in 2.0)

System requirements for CodeIgniter Wizard

  • macOS 10.13 or later
  • macOS 15 Sequoia recommended
  • Minimum screen resolution of 1440x900
  • MySQL / MariaDB 5.7 or later

System requirements for Generated PHP applications

  • PHP and Required Extensions

    PHP version 8.1 or newer is required, with the following PHP extensions are enabled:

    • intl
    • mbstring
    • json
    • mysqlnd 
    • imagick (if you use automatic image resizing following file uploads - in cases where Image class is set to use ImageMagickHandler)
    • gd (if you use automatic image resizing following file uploads - in cases where Image class is set to use GDHandler)

    Warning

    • The end of life date for PHP 7.4 was November 28, 2022.
    • The end of life date for PHP 8.0 was November 26, 2023.
    • If you are still using PHP 7.4 or 8.0, you should upgrade immediately.
    • The end of life date for PHP 8.1 will be December 31, 2025.

    Optional PHP Extensions

    The following PHP extensions should be enabled on your server:

    • curl (if you use CURLRequest)

    • simplexml (if you format XML)

  • Access to the database server as indicated in the '.env' file or app/Config/Database. (You can specify a different database than MySQL / MariaDB via custom code outside CI Wizard.)
    The database platforms below are supported by generated web apps using CodeIgniter 4.x.

    • MySQL (5.7+) via the MySQLi driver
    • PostgreSQL via the Postgre driver
    • SQLite3 via the SQLite3 driver
    • MSSQL via the SQLSRV driver (version 2005 and above only)

CodeIgniter Wizard will run on macOS 10.13 or later (it is also optimized for macOS 15 Sequoia), and it currently supports only MySQL / MariaDB for the database platform at design time. So you need to have access to a MySQL or MariaDB Server v.5.5 or later over either your local network or the Internet (SSH tunneling option not available).
Ideally, a local MySQL would be present on your Mac, however, it is not required. You can use any MySQL/MariaDB server on your network and user account to which you have direct access.

CodeIgniter Wizard currently does not create database schemas and users from scratch on your database servers which you have to do by yourself either using a free GUI tool like MySQL Workbench, Sequel Ace (aka Sequel Pro), or a premium tool such as Navicat or simply by using the command line.

Once you have created a database schema and a user with full privileges for that schema, you define the credentials in CodeIgniter Wizard by what is called "registering a database" so that it can read and analyze your schema.

Current Limitations

Single Database

While it is possbile to configure the CodeIgniter framework itself to use multiple databases / schemas, CodeIgniter Wizard currently supports one database schema per web application. If you need to develop and deploy a web application using multiple databases and some of the entity models using one database and the others another, this is possible by manually customizing a web application previously generated with the wizard. (In fact this is how the CMS of the Ozar.net website currently works). More information on such a configuration is in the CodeIgiter 4 documentation.

Single Database Table for Module

CodeIgniter Wizard has the notion of modules in which each module corresponding to one datatabase table. There can be one MVC module for one database table. If your end web application needs both a RESTful controller and non-RESTful controller dealing with the models and entities generated from a table, becuase you cannot create two modules (meaning one model, one entity, one controller and one set of list and form views per DB table), the only way to get code for both ways is to create a duplica project in CodeIgniter Wizard, and set the controller type different than that of the first one.

Referencing vs Referenced Modules (of joined tables via foreign keys)

As of version 1.6, CodeIgniter Wizard is capable of generating all related modules when you create one module from a database table with foreign key relationships to other tables. (This setting can be turned off in Preferences). However, the wizard will create all other related modules with a RESTful Resource Controller using AJAX in list views. If you'd rather related modules do not use AJAX, you would need to edit each of them one by one and uncheck RESTFul Resource controller option.

Routing

CodeIgniter Wizard no longer leaves the app/Config/Routes.php file untouched, but optionally writes simple routes for controllers and their methods, and it no longer uses the auto-routing mechanism of CodeIgniter using class names and methods as auto-generated URIs.

If you need to specify namespaces, permissions / filters for your routes, and / or group your routes, you have to do this manually by yourself, after the code is generated.

While routes can be explicitly defined in CodeIgniter framework, unlike most other frameworks, there is an automatic routing system in CodeIgniter which the wizard no longer uses as stated above in order to avoid naming collisions in complex projects.

CAUTION:

Every time, you make a change in an already generated project in CodeIgniter Wizard, Routes.php is regenerated from scratch reflecting the most up-to-date state of your modules. If you manually write or update code in the routes(.php) file, your changes will be overwritten next time you publish a module (or re-publish all or any of the existing modules).

Manually Editing the Generated Code

Be sure to make manual changes after copying the generated folder to your development environment, and it is recommended that you always work on a copy.

While you can edit the generated folder directly, each time you make a change to your app in CodeIgniter Wizard and re-publish the changes, all the controller, model, entity classes, and view files (as well as routes) will be over-written (risking you to lose your changes if you are directly editing them).
